\# Governance and Policy



\## Objective



This section focuses on governance, baseline enforcement, and policy-aligned security controls across Azure and hybrid resources.



\## Azure Policy Examples



Policies used in this lab include:



\- Require department tags on resources

\- Restrict insecure deployments

\- Require security-related configurations where applicable

\- Demonstrate deny and remediation behavior in lab scenarios



\## Defender for Cloud



Defender for Cloud is used to:



\- Review security posture

\- Surface recommendations

\- track secure score

\- highlight baseline issues on cloud resources



\## Azure Arc



Azure Arc concepts are included to demonstrate how hybrid systems can be brought into a central governance model for visibility and compliance.



\## Department-Based Policy Controls



Examples include:



\- Finance users restricted from non-business internet categories

\- HR devices restricted from removable storage usage

\- IT devices configured with stronger auditing

\- Different network paths and controls enforced by subnet and policy



\## Why this matters



This lab is intended to show that security is not only about detection.  

It also includes preventive controls, governance decisions, and policy-based enforcement across a realistic hybrid environment.
