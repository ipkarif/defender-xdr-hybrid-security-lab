\# Detection and Response



\## Objective



This section highlights how Microsoft Defender XDR, Sentinel, and Security Copilot are used together for visibility, investigation, and response in the lab.



\## Defender XDR



The lab demonstrates:



\- Device onboarding

\- Device inventory validation

\- incidents and alerts

\- attack story review

\- web content filtering examples

\- endpoint protection status checks



\## Microsoft Sentinel



Sentinel is used as the SIEM/SOAR layer for:



\- data connector configuration

\- custom KQL queries

\- analytic rules

\- incident review

\- playbook-based automation



\## Security Copilot



Security Copilot is positioned as an investigation support tool for:



\- incident summaries

\- suspicious activity review

\- investigation assistance

\- KQL generation support



\## Example Detection Use Cases



\- Failed authentication attempts

\- suspicious endpoint activity

\- policy violations

\- NSG modifications

\- department-specific alert scenarios



\## Response Examples



\- isolate a high-severity endpoint

\- notify security stakeholders

\- review correlated incidents across users, devices, and alerts

\- document remediation and hardening steps



\## Why this matters



This project is designed to demonstrate not only visibility, but also structured triage and response across Microsoft’s security ecosystem.
